FAQ

Useful questions

Clear answers for what this project is and why it exists.

What is this website for?

This is a DevOps-style portfolio landing page meant to be served inside a Docker container using Nginx.

Why use a static website for Docker practice?

Because it removes unnecessary complexity. You focus on Dockerfile, image build, container run, port mapping, and deployment flow first.

Can this be deployed to an EC2 instance later?

Yes. Build the image, move it to the server, run the container, and expose the required port through the instance and security group.

Can I edit this into a real portfolio?

Yes. Replace placeholder stats, tools, and project descriptions with your actual DevOps work and GitHub links.
